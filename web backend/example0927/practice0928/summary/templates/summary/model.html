모델(Model)

개요

  • 장고는 ORM 방식으로 데이터베이스를 연동한다.
  • 이에 필요한 클래스를 모델이라고 부른다.
  • 모델 클래스는 데이터베이스의 테이블에 대응되며, 모델을 통해 만들어진 객체는 테이블의 레코드에 대응된다. 모델에 작성한 클래스 변수는 테이블의 필드에 대응된다.

대응 체계

장고 모델 데이터베이스
클래스 테이블
객체 레코드(테이블의 행)
클래스 변수 필드(테이블의 열)

모델 작성 방법

  • 각 앱 내 models.py 파일에 작성한다.
  • django.db.models.Model 클래스를 상속하는 하위 클래스를 작성한다.
  • 테이블의 필드를 나타내기 위해 클래스 변수를 작성한다.
  • 클래스 변수의 자료형과 옵션을 적절히 작성한다.(후술).

모델 적용 방법

  • 마이그레이션을 실행하면 작성한 모델 내용에 따라 자동으로 테이블을 생성한다.
  • 테이블 이름은 앱이름_모델이름 으로 설정한다.
  • 필드 이름은 클래스 변수 이름으로 설정한다.
  • 자료형과 제약조건은 설정된 데이터베이스에 따라 자동으로 적용한다.

필드의 종류

모델에 작성할 수 있는 필드의 종류에 대해 서술.

  • IntegerField: 32비트의 정수형 필드. 모든 데이터베이스에 대해 안전한 자료형.
  • BigIntegerField: 64비트의 정수형 필드.
  • FloatField: 부동소수점 실수형 필드. 파이썬의 float 자료형과 호환.
  • DecimalField: 고정소수점 실수형 필드. 파이썬의 Decimal 클래스와 호환.
  • CharField: 짧은 문자열을 저장하는 필드. max_length 옵션을 반드시 작성해야 함.
  • TextField: 긴 문자열을 저장하는 필드. max_length 옵션을 추가할 수도 있음.
  • BoolenField: 논리형 필드. 주로 checkbox 등과 같이 사용.
  • DateField: 날짜 데이터 필드. 파이썬의 datetime.date 클래스와 호환.
  • TimeField: 시간 데이터 필드. 파이썬의 datetime.time 클래스와 호환.
  • DatetimeField: 날짜-시간 데이터 필드. 파이썬의 datetime.datetime 클래스와 호환.
  • ForeignKey: 외래키를 나타내기 위한 필드.
  • AutoField: 자동으로 증가하는 IntegerField
  • BigAutoField: 자동으로 증가하는 BigIntegerField
  • EmailField: Email 주소를 입력하는데 특화된, CharField를 상속하는 필드.
  • URLField: URL 값을 입력하는데 특화된, CharField를 상속하는 필드.
  • FilePathField: 파일 경로를 저장하는데 특화된, CharField를 상속하는 필드.
  • FileField: 파일을 저장할 때 사용하는 필드.
  • ImageField: 이미지 파일을 저장할 때 사용하는 필드.

옵션의 종류

필드를 작성할 때 추가할 수 있는 옵션. 주로 제약조건을 나타낸다.

  • primary_key: 기본키 제약조건을 추가할 때 사용하는 옵션.
  • unique: UNIQUE 제약조건을 추가할 때 사용하는 옵션.
  • null: NOT NULL 제약조건을 해제할 때 사용하는 옵션.
  • blank: 데이터를 검증하지 않는 옵션.
  • default: DEFAULT 제약조건을 추가할 때 사용하는 옵션.

null=True은 테이블에 데이터가 누락된 것을 허용하는 옵션이고, blank=True는 웹 앱에서 데이터 누락 검증을 하지 않는다는 옵션이다.
