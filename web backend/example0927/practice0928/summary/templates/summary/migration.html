마이그레이션

모델에 작성된 내용에 따라, 데이터베이스에 테이블을 생성하는 기능.

마이그레이션 작업 순서

  1. 마이그레이션 파일 생성.
  2. 마이그레이션 파일 적용.

마이그레이션 파일 생성.

모델의 내용을 DB에 적용하기 전, 적용할 내용들을 기록하는 파일을 생성하는 단계.

명령어를 실행하면 앱 내 migrations 폴더에 순차적으로 파일이 기록됨.

마이그레이션 파일을 함부로 삭제하면 마이그레이션 작업이 안될 수 있음.

ex) py manage.py makemigrations board

마이그레이션 파일 적용.

생성된 마이그레이션 파일을 데이터베이스에 적용하는 단계.

앱 이름, 파일 번호를 지정하지 않으면 모든 앱의 최신 버전의 파일에 대해 적용함.

파일 번호를 지정하지 않으면 지정된 앱의 최신 버전의 파일에 대해 적용함.

앱 이름, 파일 번호를 지정하면 지정된 앱의 지정된 번호의 파일에 대해 적용함.

ex) py manage.py migrate

ex) py manage.py migrate board

ex) py manage.py migrate board 0002

마이그레이션 파일 내역 확인

지정된 앱에 대해 모든 마이그레이션 파일의 목록과 적용 여부를 출력하는 명령어.

ex) py manage.py showmigrations board

ex) py manage.py showmigrations user board admin ...

마이그레이션 쿼리문 확인

지정된 앱의 지정된 파일 번호에 대해 실행할 쿼리문을 출력하는 명령어.

ex) py manage.py sqlmigrate board 0002
